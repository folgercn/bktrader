# 变更日志 (Changelog)

本项目的所有重大里程碑变更都将记录在此文件中。

## [2026-04]

### 2026年4月 里程碑

- **bktrader-ctl 控制面板**: 引入面向 LLM/Agent 的控制面板，作为 bktrader 的核心管理平面，支持生产环境的高级运维与状态观测。
- **实盘恢复与稳定性加固**: 
    - 实现 **Live Recovery Workbench**，支持实盘会话的运行时状态恢复、接管及被动平仓逻辑。
    - 引入运行时租约 (Runtime Lease) 与硬性对账门控，确保分布式环境下的执行唯一性与数据一致性。
- **监控架构升级 (SSE)**: 仪表盘从传统的 HTTP 轮询全面转向 **Server-Sent Events (SSE)** 流式更新，大幅降低服务器压力并提升响应实时性。
- **UI/UX 现代化 (Bento Style)**: 
    - 采用现代 **Bento 风格** 重构仪表盘与监控控制台。
    - 集成 **shadcn/ui** 组件库，统一全站视觉系统与交互规范。
- **统一日志与遥测系统**: 
    - 构建后端结构化日志 API，并配套前端实时日志查看台，实现生产环境透明化排障。
    - 增加实盘数据收集遥测与后端健康快照。
- **交易执行引擎重构**: 
    - **Live Trade Pairs 架构**: 实现交易对独立核验与平仓路径解耦。
    - 引入针对不匹配订单的 **手动交易验证 (Manual Verification)** 流程。
    - 加固止损保护与重新入场门控，对齐高频/日内策略的研究基准。
- **多渠道通知集成**: 实现 Telegram 交易事件实时分发、持仓报告以及数据源过时告警去抖。
- **算法优化与清理机制**: 
    - 引入 **止损重新入场最小延迟 (SL Reentry Min Delay)**，优化极端行情下的执行稳定性。
    - 实现 **零仓位清理 (Zero-quantity cleanup)** 与孤立运行时自动回收，保持系统环境纯净。
- **CI/CD 与运维体系**: 实现 CLI 自动发布流程，完善 Docker 隔离部署与测试网最小闭环验证。

### 即将发布 / 正在进行中

- **声明式实盘会话控制 (Declarative Control)**: 引入 `desired-state` 驱动模型，支持更稳健的会话生命周期管理。
- **成交对账引擎 v2 (Issue #272)**: 包含交易所成交报告映射、结算锁 (Settlement Lock) 以及更深层次的账务对账自愈机制。
- **运维可观测性增强 (Phase 5)**: 统一全系统错误码规范，并在 `bktrader-ctl` 中集成深度执行指标与性能监控。
