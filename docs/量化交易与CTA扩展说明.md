# bktrader 量化交易与 CTA 扩展说明

## 1. 这是什么项目

用人话说，`bktrader` 不是“预测软件”，而是一个会按照固定规则盯盘、下单、止损、对账的自动交易系统。

它的核心价值不只是“找到机会”，更重要的是：

- 把交易规则写死，减少临场拍脑袋
- 把下单、止损、对账流程自动化
- 在实盘里尽量保证执行一致性和安全性

所以它更像一个“纪律化交易底座”，而不是一个单纯给买卖建议的分析工具。

## 2. 这个项目是怎么赚钱的

这个项目的赚钱逻辑，不是靠神预测，而是靠以下流程：

1. 先等待市场出现符合规则的信号
2. 信号出现后，按策略规则分批尝试进入
3. 如果行情按预期发展，就持有并在合适位置退出获利
4. 如果行情不对，就按规则止损，把亏损控制住
5. 长期重复执行，争取做到“小亏大赚，总账为正”

更直白一点：

- 它赌的不是“每次都对”
- 它赌的是“长期下来，正确时赚的钱大于错误时亏的钱”

当前项目里的 baseline 也偏保守，不是一次性重仓压进去，而是倾向于：

- 先小仓位试探
- 再根据同一段行情确认情况做有限补仓
- 错了及时退出

这类系统能不能赚钱，关键不在自动化本身，而在于策略规则是否真的具备长期优势。

## 3. 这算不算量化交易

算，而且是比较典型的“规则型量化交易”。

可以这样理解：

- `程序化交易`：强调“由程序自动执行”
- `量化交易`：强调“由数据和规则决定如何交易”

`bktrader` 同时具备这两点，所以它属于量化交易的一类。

但它不是大众印象里那种最极端、最神秘的量化：

- 不是高频抢速度
- 不是纳秒级竞赛
- 不是黑箱 AI 自主决策

它更接近：

- 规则驱动量化
- 中低频自动化交易
- CTA / 趋势跟随风格的一类变体

## 4. 什么是 CTA / 趋势跟随风格

用最简单的话说，CTA / 趋势跟随的核心思想通常是：

- 市场走强，就尝试顺着强势方向做
- 市场走弱，就尝试顺着弱势方向做
- 不执着于抄顶抄底
- 更重视“跟随已经出现的方向”

这类策略通常不会要求每次都判断最高点和最低点，而是接受：

- 有时进场会慢一点
- 有时会被假突破骗出去
- 但只要抓住少数几段大行情，就可能覆盖很多次小亏损

从这个角度看，项目当前的 `breakout / reentry / exit` 框架，与 CTA / 趋势跟随的思想是接近的，只是实现细节和风控边界更工程化、更严格。

## 5. 现在能不能直接说“上线就赚钱”

不能直接这么说。

更准确的说法应该是：

- 目前已经有正向的研究和回测信号
- 系统的 testnet 自动下单和执行链路也已经打通
- 但这还不足以证明“正式上线后就能稳定赚钱”

原因很简单：

- 回测收益不等于实盘收益
- 实盘里还有手续费、滑点、资金费、拒单、网络延迟、交易所限制
- 市场环境也会变化，过去有效的规则未来可能失效

所以这个项目现阶段更适合表述为：

“已经具备继续验证真实交易优势的工程基础，但是否能稳定赚钱，还需要前向样本和真实资金验证。”

## 6. 后续能不能引入更多 CTA / 趋势跟随变种

能，而且从当前架构看，是适合逐步引入的。

但引入方式必须克制，不能把“新策略变种”和“实盘执行安全”混在一起改。

比较合理的原则是：

- 新变种优先落在研究层、回测层、参数层
- 尽量不要顺手改 `live execution`、`dispatch`、`reconcile` 这类高风险执行链路
- 一个变种先证明自己，再进入更靠近实盘的阶段

换句话说，策略可以继续扩，但执行安全边界应该尽量稳定。

## 7. 最值得扩展的 3 类方向

### 7.1 现有 BK 策略的小步增强版

这是最值得优先做的方向。

特点：

- 不改变主体哲学
- 仍然围绕 `breakout / reentry / exit`
- 主要优化过滤条件、仓位参数、止损止盈细节

适合尝试的点包括：

- 不同周期过滤组合
- 更细的入场确认
- 不同 `reentry_size_schedule`
- 不同 `max_trades_per_bar`
- 不同止损和 trailing 参数

优点：

- 与现有系统最兼容
- 风险最低
- 最容易复用现有回测和实盘框架

### 7.2 更标准化的趋势跟随独立版本

这是第二优先级。

可以理解为单独做一套更标准的趋势跟随策略，例如：

- 均线趋势过滤
- 突破入场
- 回撤后二次进场
- ATR 止损
- 移动止盈

优点：

- 逻辑清晰
- 容易解释
- 容易和现有策略做并行比较

注意：

- 这已经更像“新策略族”
- 不应被当成简单调参
- 应该单独研究、回测、前向验证

### 7.3 多策略组合框架

这是以后很有价值，但不建议太早做的方向。

它的思路是同时运行多种风格，例如：

- 一个偏 `1d` 趋势版本
- 一个偏 `4h` breakout 版本
- 一个更保守过滤的版本

潜在好处：

- 不把收益全部押在单一模式上
- 有机会让整体净值曲线更平滑

现实难点：

- 资金分配更复杂
- 策略之间可能竞争仓位
- 风险预算、监控、恢复、一致性都会更难

因此它更适合放在后期。

## 8. 当前建议排序

如果只按投入产出比和工程风险排序，建议是：

1. 先做现有 BK 策略的小步增强版
2. 再做更标准化的趋势跟随独立版本
3. 最后再考虑多策略组合框架

一句话总结：

当前最重要的，不是一下子发明很多新策略，而是先把“新增一个策略版本并完成验证”的流程标准化。

这份文档当前偏“用人话解释”和“方向判断”。后续如果继续深入，建议补充：

- 当前 BK baseline 的策略结构图

- 多策略组合时的资金分配与风险预算原则
- 从研究参数到实盘参数的收敛方法

## 10. 当前一句话结论

`bktrader` 当前可以理解为：

一个以执行安全和纪律一致性为核心的规则型量化交易系统；它未来完全可以扩展出更多 CTA / 趋势跟随变种，但应坚持“小步引入、逐层验证、不要扰动实盘安全边界”的原则。
