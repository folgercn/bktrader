# Bento Grid UI 重构与现代化架构踩坑指南及规范

本文档用于总结在将 BKTrader 控制台（如 `MonitorStage`）向极简、高密度的 **Bento Grid**（便当盒网格）风格重构时，所遭遇的技术踩坑记录，并为团队和后续介入的 AI Agent 沉淀最佳规范。

## 1. 背景与目标

为了优化高频交易系统的监控台视觉，提升首屏信息密度与交互质感。我们主要实施了以下变化：
1. **废弃 Accordion 堆叠**：用平面展开的 Bento Grid（非均衡异形卡片布局）替代了旧版深嵌套的折叠面板区域。
2. **重构会话栈选择器**：用顶层 ID 集成下拉选择器取代了庞杂的 Session Stack。
3. **增加高可视区组件**：增加常驻底部的全景 Execution Timeline Terminal 用于日志溯源。

但在重组 DOM 和样式对齐的重构过程中，产生了一系列深水区的陷阱。

---

## 2. 踩坑记录 (Post-Mortem)

### 坑 1：重组 DOM 时的“误杀”与子组件丢失
在极简化的过程中，剥离旧的容器外壳时（涉及大量 `<div>` 和闭合标签的更换），非常容易因为缩进或者光标识别不准，**把本来需要保留的核心指标区块（如 4 个就绪检测卡片）当作垃圾代码一并删除**。这直接导致重构后遥测信息丢失。

### 坑 2：脱离“安全护城河”导致的 TypeScript 空指针与严格类型报红
将左右分栏的一些卡片因视觉对齐原因（如将辅助信息卡片从右边下放到左侧底端）进行物理搬迁时：
- 原来右侧整体都被包裹在 `{monitorSession ? (...) : (...)}` 的绝对安全域里。
- 搬迁到左侧脱离了安全域后，对 `monitorSession.state.xxx` 的链式访如果忘记补上 `?.`，就会触发严重的 `Object is possibly 'null'` 红线报错。
- 由于 `state` 在当前领域架构中被定义为泛型字典 `Record<string, unknown>`，TypeScript 进行严格编译时拒绝用 `.` 语法调用其内部无显式定义的键，引发 IDE 满屏红线警告。

### 坑 3：“前框后管”的不匹配与硬编码截断
在前端大开大合地铺设了一个长长的 Terminal（包含原生滚动条，准备展示满屏日志）之后，发现日志永远只有 5 行。
**真相**：当年的数据派生逻辑（`derivation.ts` 中的 `buildTimelineNotes`）因为受制于旧版前端面板窄小，强行通过 `.slice(-5)` 在数据源头截断了数据以防组件被撑爆。只改动表现层不查底层逻辑就会闹出乌龙。

### 坑 4：高频系统时间戳精度的忽视及环境兼容异常
- 由于旧版日志面板不够重要，时间戳精简展示到了“分钟”。但在高频算法执行时，同一分钟甚至同一秒可能有几十次内部流转决策，如果只露到分钟，整个日志看起来就像代码死循环卡死输出一样。
- 在为 `Intl.DateTimeFormat` 增加 `.SSS` 毫秒级展示（`fractionalSecondDigits: 3`）时，由于不同开发人员本地的 TypeScript `lib` 版本差异，部分老旧运行环境识别不出这个较新的 ECMAScript 规范配置字段并报红。

### 坑 5：视觉纵深失衡的错误修复手段
左右两侧大宽幅网格结构中解决底部不对齐问题，初学者通常习惯拼命调节 `h-[300px]` 或是设置死固定高，或是加上宽泛的 `mt-8`。这最终导致多分辨率或自适应情况下必定失衡，下方留下荒芜的背景区白边。

## 3. UI 现代化与重构标准规范 (Guidelines)

> **给所有介入本系统重构的工程师与 AI Agent 的强制要求：**

### 规则 A：防跌落与组件拆离原则
> 当需对复杂的 JSX 解体时，不准直接原位替换包裹外壳。
必须将**需要保留的核心状态遍历内容**，暂时抽成独立的不可见渲染函数或放一边，等新的骨架（如 `<Card>` / `Grid`）完全生效并保证不会出现 TSX 类型语法崩溃后，再填回原业务代码内。

### 规则 B：搬家等同于类型重写
> 只要将某块渲染 DOM 从 A 区间移动到 B 区间，必须执行**环境上下文再审视**。
1. **强制补充保护网**：对所有的 Session、Store 取值补充 `?.` 可选链，绝不允许假设外部变量一定不为空。
2. **兼容性断言**：碰到因 `Record<string, unknown>` 类的数据引起的 IDE 语法服务器警告，要果断采用 `as any` （例如 `(session?.state as any)?.count`）予以放行抹平红色视觉噪音。或对环境兼容性问题（如 Intl API 偏门属性）使用 `as any` 进行脱敏。

### 规则 C：上下游一致性破壁
> **UI 变大，推导放开**。
在增加任何呈现面积、升级报表与统计图列时，必须使用 `grep` 全局搜索对应的状态源（如 `deriveXXX`），**重点排查是否存在历史遗留的 `slice/filter/maxLength`等硬编码数据锁**并予以同等释放。
> **要求绝对精度**。
凡涉及执行侧、日志侧输出，绝不能只精确到分钟，必须精确到 `second`。若系统属高频订单管理（如本系统中的阻断/过滤），优先开通 `fractionalSecondDigits: 3`（毫秒精度）。

### 规则 D：Flex 绝对压底定律 (Bento 对齐大招)
禁止在新 UI 架构中使用定长的 `mt-xx` （margin-top）去追求肉眼底部对齐。
在构建左右平衡的 Grid 列时，面对不确定长度的组件高度坑位，唯一合法、健壮的沉降下界对齐公式如下：
```tsx
// 父级容器使用 h-full 与 flex 列方向伸展
<div className="flex flex-col h-full"> 
  <div>上部内容</div>
  {/* 利用 mt-auto 霸占所有剩余流式冗余空间，将其稳稳压制在底部 */}
  <div className="mt-auto">需贴底对齐卡片组</div>
</div>
```
